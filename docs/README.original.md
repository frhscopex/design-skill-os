## 🧠 What is this?
Design is not just "making things look pretty." It is a science of communication, empathy, and strategic problem-solving. Most AI agents produce generic, uninspired design because they lack a **specialized cognitive framework**.

The **Design Skill OS** is a high-performance cognitive overlay for AI. It synthesizes the wisdom of **10 design bibles** (Millman, Wheeler, Lupton, etc.) into a cohesive system that guides the AI to think, research, and execute like a Master Designer.

## 🚀 Installation Guide

### Step 1: Install Globally
Get the official Design Skill CLI installed on your machine:
```bash
npm install -g design-skill-os
```

### Step 2: Project Initialization
Navigate to your project folder and initialize the skill environment:
```bash
design-skill init
```
This deploys the `SKILL.md` file (the "Design Brain") into your project workspace.

### Step 3: AI Activation
Run the activation command to get your elite system instruction:
```bash
design-skill prompt
```
Copy the output and paste it into your AI agent's system prompt or the start of your chat.

## ⚖️ The Definitive Difference (Detail)

Why use this skill? Here is a deep-dive comparison into the **"Master vs. Mid"** performance gap.

### 🎭 1. The Discovery & Strategy Phase
*   **Without the Skill (Default AI):**
    - The AI starts with code immediately. 
    - It asks surface-level questions like *"What colors do you like?"* 
    - Output: A functional website that feels like a generic template. It has no soul, no target audience focus, and no unique selling point.
*   **With the Design Skill OS:**
    - The AI **refuses to design** until it completes a "Discovery Intake" (Section 21). 
    - it asks high-impact questions: *"If this brand were a person, what would move them to action?"* 
    - Result: It builds a **Brand Manifesto** first. Every color and font choice is now a strategic weapon backed by design history.

### 🏗 2. Visual Hierarchy & Gestalt Logic
*   **Without the Skill:**
    - Uses standard 12-column grids with equal spacing. 
    - Information is just "dumped" on the page. 
    - Output: Users feel overwhelmed or bored. There is no clear journey for the eye.
*   **With the Design Skill OS:**
    - Applies **Gestalt Principles** (Proximity, Continuity, Closure) from Section 5. 
    - Uses **Narrative Storytelling** (Section 8) to create a "Call to Adventure." 
    - Result: The user's eye is mathematically guided to the "Buy Now" or "Sign Up" button. The site feels "premium" because of its intentional whitespace and modular scale.

### 🎨 3. Chromatic & Typographic Authority
*   **Without the Skill:**
    - Uses random Google Fonts (like Roboto/Open Sans) and generic palette generators. 
    - Output: Looks like 90% of the internet. It lacks authority.
*   **With the Design Skill OS:**
    - **Typography:** Uses authority-pairings (Section 3) based on the "Authority vs. Empathy" matrix. 
    - **Color:** Applies the **60-30-10 distribution** using shades chosen for their psychological weight (Section 4). 
    - Result: The site feels like a **proprietary corporate identity**, not a "built with AI" landing page.

### 🛠 4. Quality Control & Red-Teaming
*   **Without the Skill:**
    - The AI gives you the first thing it thinks of. 
    - Output: Contains "lazy" design patterns and accessibility errors.
*   **With the Design Skill OS:**
    - The AI runs its own **Self-Critique Protocol** (Section 23). 
    - It "Red Teams" the design, asking: *"Does this incite action? Is this just a schtick?"* 
    - Result: You get the **3rd or 4th iteration** of the idea, polished to professional standards before you even see it.
